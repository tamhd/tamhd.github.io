DEMO

Here you can find my personal products, all of which are associated with NLP.


vi-accenter

 Vietnamese Accent Restoration
I wrote this program a while ago. It was based on Hidden Markov Model (HMM).

My current server is shamefully weak, so the program is a bit slow :( The live demo is no longer available due to me being lazy and not maintaining the server.

When I was doing research in Machine Translation, it was difficult to build a decent model without sufficient parallel dataset, so I wrote a tool to triagulate via a third language. It was compatible with Moses.

EXPERIENCE

My daily job used to involve big data, Machine Learning and occasionally production.

MT

Machine Translation
The task when non-MT people constantly tell me: "Why bother doing it, useless, you would never beat Google Translate".

I was heavily involved in Statistical MT for around 2 years, primarily phrase-based approaches. Now the field is all about Neural MT (exited!).

GEC

Grammatical Error Correction

Research into the possibility of automatically correct written English. Mainly, it was formulated as 'translating erroneous input into proper English' (thanks to MT).

Competing with state-of-the-art solution and tier-1 NLP groups worldwide was honestly thrilling.

Fraud Detection

Fraudulent activities are critical problems of every internet company, ranging from bank to e-commerce. Each has their own definition. I was lucky to be involed in not only machine learning modeling, system production but also operational activities.

Fun fact: unlike the other projects, it is not NLP!


Sentiment Analysis


Sentiment Analysis, or Opinion mining, trying to guess whether the text has positive or negative connotation. My project targeted the problem of document-level sentiment analysis with IMDB movie reviews.

It is in fact a very important tasks nowadays regarding news, feedback, reviews, etc.

IR

Information Retrieval


It is about giving users the most relevant documents. Not only is IR an important field on its own, but it also helps other task.

I came across IR not one, not two, but three times. Each with a different solution: string edit distance, vector space model and Lucene/Solr library.

QA

Question Answering


QA tries to answer the question in customer's mind. As part of NLU, it is one of the most difficult topics in NLP.

From 2010 to 2012, we relied on rule-based (hand-crafted GATE regex macro) and knowledge-based (Ontology).
